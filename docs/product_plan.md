# H5 TODO 管理系统产品策划案

## 1. 项目概述

本项目旨在开发一个完整的 H5 TODO 管理系统，提供移动端优先的用户体验。系统包含日历视图、快速添加任务、事件编辑、数据持久化等核心功能，采用响应式设计，确保在主流移动浏览器上的兼容性。

## 2. 功能模块划分

### 2.1 日历视图模块 (Calendar View) [已完成]

- **月视图/周视图切换**：用户可根据需要切换查看模式。
- **周视图默认**：打开应用默认显示周视图，更适合移动端屏幕尺寸。
- **每日事件标识**：在日历日期下方显示小圆点，标识当天待办事项数量。
- **日期详情**：点击具体日期，下方列表展示该日期的所有 TODO 事项。
- **滑动切换**：支持左右切换月份或周。
- **今天按钮**：快速回到当前日期。

### 2.2 快速添加模块 (Quick Add) [已完成]

- **输入入口**：首页底部放置快速输入框。
- **交互**：
  - 点击输入框唤起键盘。
  - 输入内容后按回车键（或确认键）快速提交。
  - 默认添加到当前选中的日期。
  - 默认不绑定分类（无分类），可在编辑页面手动选择。
  - 提交成功后触发平滑过渡动画。

### 2.3 事件编辑模块 (Event Edit) [已完成]

- **基础信息**：
  - 标题（必填，大字号无边框设计）
  - 内容/备注（选填，气泡式设计）
- **时间设置**：
  - 开始时间、结束时间（支持精确到分钟）
  - 全天事件开关
- **高级设置**：
  - **重复周期**：支持不重复、每天、每周、每月、每年。
  - **子任务清单**：支持添加/删除/完成子任务，带进度显示。
  - **提醒时间**：支持无提醒至提前 1 天。
  - **优先级**：高、中、低（颜色区分）。
  - **分类标签**：支持选择现有分类或新建分类（颜色标识）。
  - **主题关联**：支持关联自定义主题。
- **删除确认**：删除任务时显示二次确认弹窗，防止误操作。

### 2.4 数据存储模块 (Data Storage) [已完成]

- **本地存储**：使用 `localStorage` 进行数据持久化。
- **状态管理**：Zustand Store 管理 Tasks, Categories, Themes, SortOptions。

### 2.5 响应式布局与动画 [已完成]

- **移动端优先**：Tailwind CSS 适配。
- **动画系统**：集成 Framer Motion，实现模态框进出、列表排序、页面切换等 60fps 流畅动画。

### 2.6 分类与主题管理 [已完成]

- **主题管理**：支持自定义主题色、关联优先级。
- **分类管理**：支持创建、删除分类，自定义分类颜色。
- **删除确认**：删除分类/主题时显示二次确认弹窗，关联任务自动解除绑定。
- **归档功能**：支持将分类/主题归档，已归档的项目不在编辑界面显示，支持取消归档恢复使用。

### 2.7 通用确认弹窗模块 (Confirm Dialog) [新增/已完成]

- **功能**：统一的删除确认弹窗组件。
- **使用场景**：
  - 删除任务
  - 删除子任务
  - 删除分类
  - 删除主题
- **特性**：
  - 统一的视觉风格（红色警告图标）
  - 支持自定义标题、消息、按钮文字
  - 支持 `danger` 和 `primary` 两种按钮样式
  - 流畅的动画效果

## 3. 用户流程设计 (User Flow)

1.  **查看任务**：
    首页 -> 日历视图（默认周视图）-> 点击日期 -> 任务列表（支持排序/过滤） -> 点击任务卡片 -> 任务编辑页。

2.  **快速添加**：
    首页 -> 底部输入框 -> 输入内容 -> 回车 -> 列表自动刷新并显示新任务（默认无分类）。

3.  **管理分类/主题**：
    首页顶部 -> 点击文件夹/调色盘图标 -> 打开管理弹窗 -> 新增/删除（带确认） -> 关闭。

4.  **删除任务**：
    任务列表/编辑页 -> 点击删除按钮 -> 显示确认弹窗 -> 确认删除 -> 任务移除。

## 4. 技术选型方案

- **核心框架**：React 18
- **构建工具**：Vite
- **语言**：TypeScript
- **样式库**：Tailwind CSS
- **图标库**：Lucide React
- **动画库**：Framer Motion
- **日期处理**：date-fns
- **状态管理**：Zustand
- **测试框架**：Vitest

## 6. 设计规范要点

### 6.1 交互设计

- **操作按钮常驻**：所有编辑/删除按钮常驻显示，不依赖 hover 状态
- **删除二次确认**：所有删除操作必须经过确认弹窗
- **默认视图**：日历默认周视图，更适合移动端

### 6.2 视觉设计

- **确认弹窗**：统一使用红色警告图标 + 白色背景
- **按钮层次**：删除按钮红色系，编辑按钮蓝色系
- **动画统一**：所有弹窗使用 animations.ts 中的动画常量
