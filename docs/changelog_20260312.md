# 更新日志 - 2026-03-12

## 排序功能优化

### 新增功能

1. **按截至时间排序**
   - 支持按任务截至时间升序/降序排列
   - 相同时间的任务按优先级进行二次排序（高 > 中 > 低）

2. **按分类排序**
   - 支持按分类名称升序/降序排列
   - 相同分类的任务按优先级进行二次排序
   - 实现分组显示功能，相同分类的任务归为一组
   - 支持分组折叠/展开，点击分组头部可切换显示状态

3. **按主题排序**
   - 支持按主题名称升序/降序排列
   - 相同主题的任务按优先级进行二次排序
   - 实现分组显示功能，相同主题的任务归为一组
   - 支持分组折叠/展开，点击分组头部可切换显示状态

### 技术实现

#### TaskList 组件重构

- **新增 TaskItem 子组件**: 将任务卡片抽离为可复用的子组件，支持紧凑模式 (compact)
- **分组逻辑**: 使用 `useMemo` 缓存分组结果，避免不必要的重新计算
- **折叠状态管理**: 使用 `Set` 数据结构管理折叠的分组 ID
- **排序优化**: 所有排序方式均支持优先级二次排序

#### 性能优化

- 使用 `useMemo` 缓存 `daysTasks`、`groupedTasks` 和 `sortedTasks`
- 避免在渲染过程中创建新的数组或对象引用
- 使用 Framer Motion 实现流畅的列表动画

### UI/UX 改进

#### 分组头部设计

- 显示分类/主题图标（Folder / Palette）
- 使用分类/主题的颜色作为标签背景
- 显示组内任务数量
- 折叠/展开箭头带有旋转动画

#### 动画效果

- 分组折叠/展开使用 height + opacity 动画
- 任务卡片使用 Framer Motion 的 layout 动画
- 过渡时间控制在 200ms，快速响应用户操作

### 代码规范

- 遵循项目编码规范，使用 `import type` 导入类型
- 复用 `animations.ts` 中的动画常量
- 使用 Tailwind CSS 的 Design Token 进行样式设置
