## 2. 自定义日期时间选择器 (DateTimePicker)

### 2.1 背景

为避免使用浏览器原生 `<input type="date">` 和 `<input type="datetime-local">` 组件，确保视觉风格与项目完全一致，开发了自定义日期时间选择器。

### 2.2 新增文件

**文件**: `src/components/DateTimePicker.tsx`

### 2.3 功能特性

- **日历视图**: 支持年/月/日三级选择
- **时间选择**: 支持时分选择（非全天事件）
- **动画效果**: 使用 `animations.ts` 中的动画常量
- **响应式设计**:
  - 移动端: 底部弹出全屏模态框
  - 桌面端: 下拉面板

### 2.4 视觉规范

| 属性          | 值                                                    |
| ------------- | ----------------------------------------------------- |
| 弹窗圆角      | `rounded-t-2xl` (移动端) / `rounded-2xl` (桌面端)     |
| 选中状态      | `bg-blue-600 text-white shadow-md shadow-blue-500/25` |
| 按钮点击反馈  | `active:scale-[0.98]`                                 |
| 年份/月份选择 | 网格布局，选中项高亮                                  |

### 2.5 在 TaskEditModal 中的使用

替换原有的原生日期时间输入框：

```tsx
// 修改前
<input type={isAllDay ? "date" : "datetime-local"} ... />

// 修改后
<DateTimePicker
  value={startDate}
  onChange={setStartDate}
  isAllDay={isAllDay}
  label="开始时间"
/>
```

## 3. 自定义下拉选择组件 (CustomSelect)

### 3.1 背景

为避免使用浏览器原生 `<select>` 组件，开发了自定义下拉选择组件，支持图标、颜色标识等丰富选项展示。

### 3.2 新增文件

**文件**: `src/components/CustomSelect.tsx`

### 3.3 功能特性

- **选项支持**: 图标、颜色标识、文字标签
- **选中状态**: 蓝色高亮 + 勾选图标
- **动画效果**: 下拉列表使用 `listItemVariants`
- **响应式设计**:
  - 移动端: 底部弹出模态框（类似原生选择器）
  - 桌面端: 下拉列表

### 3.4 视觉规范

**桌面端下拉列表**:
| 属性 | 值 |
|------|-----|
| 圆角 | `rounded-xl` |
| 选中背景 | `bg-blue-50` |
| 选中文字 | `text-blue-700` |
| 勾选图标 | `Check` 图标，蓝色 |

**移动端底部弹窗**:
| 属性 | 值 |
|------|-----|
| 弹窗圆角 | `rounded-t-2xl` |
| 遮罩层 | `bg-black/40 backdrop-blur-[2px]` |
| 标题栏 | 标题 + 关闭按钮 |
| 选项高度 | `py-4`（更大触控区域） |
| 字体大小 | `text-base` |
| 动画 | 列表项依次滑入（stagger delay: 0.03s） |

### 3.5 在 TaskEditModal 中的使用

替换原有的原生 select 元素：

```tsx
// 重复规则选择
<CustomSelect
  value={repeatRule}
  onChange={(value) => setRepeatRule(value as RepeatRule)}
  options={[...]}
  icon={<Repeat size={16} />}
/>

// 提醒时间选择
<CustomSelect
  value={reminderTime}
  onChange={(value) => setReminderTime(value as ReminderTime)}
  options={[...]}
  icon={<Bell size={16} />}
/>

// 主题选择
<CustomSelect
  value={themeId || ''}
  onChange={(value) => setThemeId(value || undefined)}
  options={[...]}
  icon={<Tag size={16} className="text-purple-500" />}
/>

// 分类选择
<CustomSelect
  value={categoryId || ''}
  onChange={(value) => {...}}
  options={[...]}
  icon={<Folder size={16} className="text-blue-500" />}
/>
```

## 4. 状态管理调整

### 4.1 日期状态类型变更

**文件**: `src/components/TaskEditModal.tsx`

| 状态      | 修改前   | 修改后              |
| --------- | -------- | ------------------- |
| startDate | `string` | `Date`              |
| endDate   | `string` | `Date \| undefined` |

### 4.2 初始化逻辑调整

```tsx
// 修改前
const inputFormat = task.isAllDay ? "date" : "datetime";
setStartDate(formatForInput(task.startDate, inputFormat));
setEndDate(task.endDate ? formatForInput(task.endDate, inputFormat) : "");

// 修改后
setStartDate(new Date(task.startDate));
setEndDate(task.endDate ? new Date(task.endDate) : undefined);
```

### 4.3 保存逻辑调整

```tsx
// 修改前
startDate: new Date(startDate).toISOString(),
endDate: endDate ? new Date(endDate).toISOString() : undefined,

// 修改后
startDate: startDate.toISOString(),
endDate: endDate?.toISOString(),
```

## 5. 移动端适配优化

### 5.1 问题识别

初始的 CustomSelect 移动端实现只是简单的下拉列表，存在以下问题：

- 选项区域过小，触控困难
- 与原生移动应用的选择体验差异大
- 缺乏标题说明，用户可能不清楚当前操作

### 5.2 优化方案

采用底部弹窗（Bottom Sheet）形式，类似 iOS/Android 原生选择器：

| 优化项   | 优化前       | 优化后              |
| -------- | ------------ | ------------------- |
| 展示形式 | 下拉列表     | 底部弹窗            |
| 弹窗圆角 | `rounded-xl` | `rounded-t-2xl`     |
| 标题栏   | 无           | 新增标题 + 关闭按钮 |
| 选项高度 | `py-2.5`     | `py-4`              |
| 字体大小 | `text-sm`    | `text-base`         |
| 图标大小 | 16px         | 20px                |
| 动画效果 | 简单淡入     | 列表项依次滑入      |
| 最大高度 | `max-h-64`   | `max-h-[70vh]`      |

| ------------ | -------------- | --------------------- | ------------ |

1. **自定义选择组件规范**

2. **日期时间选择规范**

   - 移动端优先的交互设计
   - 触控区域最小 44px

## 11. 注意事项

### 11.1 浏览器兼容性

- `DateTimePicker` 使用 `date-fns` 处理日期，兼容性良好
- 动画使用 `framer-motion`，支持现代浏览器
- 移动端底部弹窗使用 `fixed` 定位，需测试 iOS Safari 兼容性

### 11.2 性能考虑

- 日期计算在组件内部完成，避免频繁重渲染
- 动画使用 GPU 加速的 transform 和 opacity
- 选项列表使用 `AnimatePresence` 实现高效动画

### 11.3 用户体验

- 移动端底部弹窗提供更大的触控区域
- 选中状态清晰可见（蓝色高亮 + 勾选图标）
- 关闭按钮方便用户取消操作
